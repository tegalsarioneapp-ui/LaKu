Kamu adalah developer senior. Kerjakan aplikasi **BOP RT 005 Offline Manager** dengan teliti dan tuntas.

Saya ingin aplikasi ini memiliki fitur **Editor Dokumen Pro langsung di dalam aplikasi**, seperti pengalaman Google Docs / Office Word sederhana, agar hasil generate dokumen bisa diedit langsung sebelum dicetak.

Tujuan akhirnya:

**Data aplikasi → Generate dokumen → tampil di editor pro → bisa edit layout/paragraf/tabel → simpan sebagai format baku aplikasi → cetak langsung dari aplikasi.**

## TAMPILAN EDITOR

* background abu-abu muda seperti Google Docs;
* lembar putih A4 di tengah;

* margin visual;
* zoom 75%, 90%, 100%, 125%;
* scroll nyaman;
* hasil tetap terbaca.

Ukuran lembar:

* A4 portrait;
* ukuran visual konsisten;

## ENGINE EDITOR

Gunakan salah satu editor yang stabil:

* TipTap / ProseMirror, atau
* Lexical, atau

Jika aplikasi harus offline, vendor/bundle library ke dalam project. Jangan bergantung pada CDN online.

Jika project belum memakai bundler, boleh tetap static HTML/JS, tetapi library editor harus disimpan lokal di folder aplikasi.

## FITUR SIMPAN SEBAGAI FORMAT BAKU APLIKASI

Ini bagian paling penting.

Setelah saya edit hasil generate dan layout sudah sesuai, saya ingin klik:

**Simpan Sebagai Format Baku Aplikasi**

Maka aplikasi harus menyimpan format dokumen tersebut sebagai template baku untuk jenis dokumen tersebut.

Contoh:

Jika saya sedang mengedit **Surat Permohonan**, lalu saya klik **Simpan Sebagai Format Baku**, maka untuk generate Surat Permohonan berikutnya aplikasi memakai format yang sudah saya edit.

Simpan data template ke localStorage atau IndexedDB dengan key yang jelas, contoh:

`bop_rt005_template_surat_permohonan`
`bop_rt005_template_rap_1_tahun`
`bop_rt005_template_berita_acara`
`bop_rt005_template_daftar_hadir`
`bop_rt005_template_sptjm`

Format baku yang disimpan harus memuat:

* struktur HTML dokumen;
* style utama;
* placeholder data;
* pengaturan margin;
* pengaturan tabel;
* pengaturan tanda tangan;
* tanggal terakhir disimpan;
* versi template.

Tambahkan juga tombol:

* Backup Format Baku
* Import Format Baku
* Reset Format Baku

Backup harus menghasilkan file `.json`.

Import harus bisa membaca file `.json` dan memulihkan template.

## PLACEHOLDER DATA

Saat menyimpan format baku, jangan menyimpan data statis saja. Gunakan placeholder agar data bisa berubah otomatis.

Contoh placeholder:

`{{nomor_surat}}`
`{{tanggal_surat}}`
`{{nama_lurah}}`
`{{rt}}`
`{{rw}}`
`{{nama_ketua_rt}}`
`{{nama_sekretaris_rt}}`
`{{nama_bendahara_rt}}`
`{{nama_ketua_rw}}`
`{{nominal_bop}}`
`{{nama_bank}}`
`{{nama_pemilik_rekening}}`
`{{nomor_rekening}}`
`{{no_ktp_ketua_rt}}`

Untuk tabel dinamis gunakan blok khusus, misalnya:

`{{#rap_items}}`
`{{no}}`
`{{uraian_kegiatan}}`
`{{satuan_volume}}`
`{{rencana_anggaran}}`
`{{keterangan}}`
`{{/rap_items}}`

Untuk peserta:

`{{#peserta_rapat}}`
`{{no}}`
`{{nama}}`
`{{jabatan}}`
`{{ttd_nomor}}`
`{{/peserta_rapat}}`

Saat dokumen digenerate, placeholder diganti dengan data terbaru dari aplikasi.

## PRINT / PDF

Fungsi cetak harus mengambil isi dari Editor Dokumen Pro, bukan dari halaman aplikasi penuh.

Saat klik **Cetak / Simpan PDF**:

* hanya lembar dokumen yang dicetak;
* sidebar tidak ikut;
* toolbar tidak ikut;
* panel generate tidak ikut;
* hasil A4;
* logo tidak membesar;
* tabel tidak terpotong;
* font Arial;
* layout sama dengan yang terlihat di editor.

Gunakan CSS print khusus:

* `@page { size: A4; margin: ... }`
* `.document-page` hanya yang dicetak;
* semua panel UI disembunyikan saat print;
* gambar/logo punya ukuran tetap;
* table width 100%;
* border-collapse collapse;
* page-break rules rapi.
